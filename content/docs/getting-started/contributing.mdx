## Overview

Checkmate is open source and welcomes contributions from developers of all experience levels. The project follows a gitflow workflow with pull request reviews.

## Before you start

- Read through the [Checkmate repository](https://github.com/bluewave-labs/checkmate) to understand the codebase
- Join the [Discord community](https://discord.gg/NAb6H3UTjK) to connect with other contributors
- Check [open issues](https://github.com/bluewave-labs/checkmate/issues) for tasks labeled `good first issue`

## Contribution workflow

1. **Fork the repository** — create your own copy on GitHub
2. **Create a feature branch** from the main branch with your changes
3. **Push the branch** to your fork on GitHub
4. **Open a pull request** for the team to review
5. **Respond to feedback** — the team may approve or request revisions
6. **Make adjustments** if needed and push updates to the same branch
7. **Maintainer merge** — once approved, a maintainer will merge your branch

<Callout type="info">
If you're new to Git or need a refresher, check out the [Git Quick Start Guide](https://github.com/bluewave-labs/bluewave_collaborative_git) maintained by the Bluewave team.
</Callout>

## Getting the development environment running

See the [installation guide](/docs/getting-started/installation) for developer setup instructions with hot-reload support.

## Guidelines

- Follow existing code conventions and patterns in the codebase
- Write clear commit messages describing what changed and why
- Keep pull requests focused — one feature or fix per PR
- Test your changes locally before submitting
- Be respectful and constructive in code reviews

## Other ways to contribute

- **Report bugs** — open an issue on [GitHub](https://github.com/bluewave-labs/checkmate/issues)
- **Suggest features** — start a discussion on [GitHub Discussions](https://github.com/bluewave-labs/checkmate/discussions)
- **Translate** — help translate Checkmate into your language on [POEditor](https://poeditor.com/join/project/lRUoGZFCsJ)
- **Improve docs** — submit PRs to improve documentation
- **Share** — star the repo, write about Checkmate, or recommend it to others
